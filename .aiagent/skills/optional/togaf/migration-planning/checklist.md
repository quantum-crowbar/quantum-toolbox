# Migration Planning Checklist

Quick reference for TOGAF Phase F deliverables.

---

## Phase Entry Criteria

- [ ] Phase E (Opportunities and Solutions) approved
- [ ] Transition architectures defined
- [ ] Work package portfolio documented
- [ ] Dependency analysis complete
- [ ] Solution building blocks identified
- [ ] Stakeholder buy-in for approach

---

## Transition Architecture Checklist

### Transition Specification

For each transition:

- [ ] Unique ID assigned (TA-nnn)
- [ ] Clear theme/name
- [ ] Duration estimated
- [ ] Target date set

### Scope Definition

- [ ] Business changes documented
- [ ] Data changes documented
- [ ] Application changes documented
- [ ] Technology changes documented
- [ ] Clean boundaries (no partial implementations)

### Dependencies

- [ ] Prerequisites identified
- [ ] What this transition enables documented
- [ ] External dependencies captured
- [ ] No circular dependencies

### Success Criteria

- [ ] Measurable exit criteria defined
- [ ] Validation approach documented
- [ ] Stakeholder sign-off requirements clear

---

## Implementation Plan Checklist

### Structure

- [ ] Executive summary written
- [ ] Strategic context documented
- [ ] Vision reference included

### Portfolio View

- [ ] All projects listed
- [ ] Projects mapped to transitions
- [ ] Dependencies between projects shown
- [ ] Critical path identified

### Timeline

- [ ] Start/end dates for all projects
- [ ] Key milestones defined
- [ ] Milestone exit criteria clear
- [ ] Calendar constraints considered
  - [ ] Holiday periods
  - [ ] Business cycles
  - [ ] Regulatory deadlines

### Governance

- [ ] Decision authority defined
- [ ] Escalation path clear
- [ ] Reporting cadence set
- [ ] Change control process defined

---

## Project Charter Checklist

For each project:

### Identity

- [ ] Unique ID assigned (PRJ-nnn)
- [ ] Sponsor identified
- [ ] Project manager assigned or process to assign
- [ ] Work packages linked

### Business Case

- [ ] Problem statement clear
- [ ] SMART objectives defined
- [ ] Benefits identified and measurable
- [ ] Strategic alignment shown

### Scope

- [ ] In scope clearly listed
- [ ] Out of scope explicitly stated
- [ ] Assumptions documented with impact if wrong
- [ ] Constraints documented with implications

### Timeline

- [ ] Phases defined
- [ ] Milestones set
- [ ] Deliverables per phase listed
- [ ] Total duration reasonable

### Resources

- [ ] Roles identified
- [ ] FTE requirements estimated
- [ ] Budget broken down by category
- [ ] Contingency included

### Risk

- [ ] Key risks identified
- [ ] Mitigation actions defined
- [ ] Risk owners assigned

### Approval

- [ ] Success criteria defined
- [ ] Approval authorities listed
- [ ] Sign-off obtained

---

## Resource Estimation Checklist

### People

- [ ] All required roles identified
- [ ] FTE counts estimated per role
- [ ] Skills requirements documented
- [ ] Availability verified
- [ ] Gaps identified with mitigation

### Budget

- [ ] Personnel costs calculated
- [ ] Technology costs included
  - [ ] Infrastructure
  - [ ] Licenses
  - [ ] Tools
- [ ] External services quoted
- [ ] Training costs included
- [ ] Contingency calculated

### Estimation Quality

- [ ] Estimation method documented
- [ ] Assumptions listed
- [ ] Confidence level stated
- [ ] Basis for estimates provided
- [ ] Peer review completed

---

## Risk Assessment Checklist

### Risk Identification

- [ ] Technical risks captured
  - [ ] Integration complexity
  - [ ] Data migration
  - [ ] Performance
  - [ ] Security
- [ ] Operational risks captured
  - [ ] Business disruption
  - [ ] Support readiness
  - [ ] Knowledge transfer
- [ ] Organizational risks captured
  - [ ] Change resistance
  - [ ] Resource availability
  - [ ] Stakeholder commitment
- [ ] External risks captured
  - [ ] Vendor dependencies
  - [ ] Regulatory changes
  - [ ] Market conditions

### Risk Assessment

- [ ] Probability rated (1-5)
- [ ] Impact rated (1-5)
- [ ] Score calculated
- [ ] Severity categorized

### Risk Response

- [ ] Response strategy selected
  - [ ] Avoid
  - [ ] Mitigate
  - [ ] Transfer
  - [ ] Accept
- [ ] Actions defined
- [ ] Owners assigned
- [ ] Due dates set

### Risk Monitoring

- [ ] Early warning signs identified
- [ ] Review cadence established
- [ ] Escalation triggers defined

---

## Rollback Planning Checklist

### For Each Critical Component

- [ ] Migration type documented
- [ ] Rollback window defined
- [ ] RTO/RPO established

### Triggers

- [ ] Automatic rollback triggers defined
- [ ] Decision-based triggers documented
- [ ] Decision authority assigned

### Procedure

- [ ] Step-by-step procedure documented
- [ ] Time estimates for each step
- [ ] Verification steps included
- [ ] Owner for each step assigned

### Prerequisites

- [ ] Backup strategy documented
- [ ] Backup verification tested
- [ ] Rollback scripts prepared
- [ ] Scripts tested in non-prod

### Point of No Return

- [ ] PONR criteria defined
- [ ] Post-PONR strategy documented
- [ ] Stakeholders aware of PONR

### Communications

- [ ] Contact list current
- [ ] Backup contacts identified
- [ ] Communication templates ready
- [ ] Escalation path clear

---

## Stakeholder Approval Checklist

### Approval Package

- [ ] Implementation plan summary
- [ ] Project portfolio overview
- [ ] Resource requirements summary
- [ ] Risk assessment summary
- [ ] Timeline visualization

### Reviews

- [ ] Architecture Board review
  - [ ] Vision alignment confirmed
  - [ ] Technical feasibility approved
  - [ ] Standards compliance verified

- [ ] Finance review
  - [ ] Budget justified
  - [ ] Funding approved
  - [ ] Contingency appropriate

- [ ] PMO review
  - [ ] Resources available
  - [ ] No portfolio conflicts
  - [ ] Dependencies manageable

- [ ] Sponsor approval
  - [ ] Business case valid
  - [ ] Timeline acceptable
  - [ ] Success criteria agreed

### Documentation

- [ ] Approval decisions recorded
- [ ] Conditions documented
- [ ] Action items tracked

---

## Phase Exit Criteria

- [ ] Implementation plan approved
- [ ] All transition specifications complete
- [ ] All project charters approved
- [ ] Resource plan confirmed
- [ ] Risk assessment accepted
- [ ] Rollback plans documented
- [ ] Governance approval obtained
- [ ] Ready to proceed to Phase G

---

## Quick Reference: Common Issues

| Issue | Check |
|-------|-------|
| Unrealistic timeline | Compare to analogous projects |
| Budget overrun risk | Verify contingency percentage |
| Resource conflicts | Check portfolio dependencies |
| Missing risks | Use standard risk checklist |
| Unclear scope | Review in/out of scope lists |
| Approval delays | Confirm authority and process |
