## How to Use

### When to Read These Files

1. **Session Start** - Read all files at the beginning of each new coding session
2. **Before Major Tasks** - Re-read relevant sections before starting significant work
3. **After Pulling Changes** - Check for updates to these files after git pull
4. **When Uncertain** - Refer back to instructions when unsure about conventions

### Reading Order

1. Start with this file (AGENTS.md) for overview
2. Read context.md to understand the current project state
3. Read instructions.md for coding standards and allowed operations
4. Read workflows.md for the specific task you're about to perform

### Updating These Files

- **context.md** - Update frequently as the project evolves
- **instructions.md** - Update when adding new tools or changing standards
- **workflows.md** - Update when processes change
- **glossary.md** - Update when new terminology is introduced

## Overview

AI agents should read all linked documents before starting work on this project. Each document serves a specific purpose:

1. **Instructions** - Rules that don't change frequently (coding standards, security policies, allowed tools)
2. **Context** - Project-specific information that evolves as development progresses
3. **Workflows** - Step-by-step processes for common development tasks
4. **Glossary** - Definitions of project-specific terms and acronyms
5. **Templates** - Standardized formats for commits, PRs, and documentation
