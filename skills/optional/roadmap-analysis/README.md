# Roadmap Analysis Skill

> **Audience:** AI Agents
> **Tier:** Optional
> **Invocation:** `"Analyze roadmap complexity"` / `"Expand initiative [N]"` / `"Plan team for roadmap"`

Deep-dive analysis and enrichment of an existing roadmap. Works on roadmaps generated by the `roadmap-building` skill or on any roadmap provided by the user.

---

## Purpose

Where `roadmap-building` produces the roadmap, `roadmap-analysis` goes deeper: expanding initiatives into step-by-step plans, identifying unknowns that need research, quantifying complexity, sizing the team, and producing stakeholder-ready exports.

---

## When to Use

- After `roadmap-building` to enrich specific initiatives
- When the team needs detailed implementation plans before starting
- When stakeholders need complexity assessment or team/cost estimates
- When you need to validate assumptions before committing to a timeline
- When a roadmap needs to be exported for external consumption (PDF, presentation)

---

## Input Sources

### Option A: Generated Roadmap (preferred)

```
roadmaps/
├── master-roadmap.md
├── roadmap-tracker.md
└── initiatives/
    ├── initiative-001-<name>.md
    └── initiative-002-<name>.md
```

### Option B: External Roadmap

Any format the user provides:
- Markdown files
- JIRA/Linear export (copy-paste or file)
- Gantt chart description
- Spreadsheet content
- Plain text description

Agent parses and structures before analysing.

---

## 7 Analysis Modes

### Mode 1: Expand

Take a high-level initiative and produce a detailed, step-by-step implementation plan with prerequisites, rollout sequence, and concrete tasks.

**Invocation:** `"Expand initiative [N]"` / `"Detail the API Gateway initiative"`

**Output:** `roadmaps/analysis/plans/plan-NNN-<name>.md`

---

### Mode 2: Research

Identify and document research questions within the roadmap — technology comparisons, performance benchmarks, approach validation. Produces structured research briefs that define scope, methodology, timeline, and decision impact.

**Invocation:** `"Identify research needs"` / `"Create research plan for initiative [N]"`

**Output:** `roadmaps/analysis/research/research-NNN-<topic>.md`

---

### Mode 3: Complexity Assessment

Score each initiative by technical complexity, organisational complexity, and risk. Produce an overall roadmap complexity report with confidence ratings, bottleneck analysis, and critical path.

**Invocation:** `"Analyze roadmap complexity"` / `"Complexity report"`

**Output:** `roadmaps/analysis/complexity-analysis.md`, `roadmaps/analysis/metrics/complexity-metrics.json`

---

### Mode 4: Spike Identification

Find the highest-uncertainty assumptions in the roadmap and convert them into time-boxed technical spikes — experiments designed to produce a Go/No-Go decision within a fixed duration.

**Invocation:** `"Identify spikes"` / `"What needs a proof of concept?"`

**Output:** `roadmaps/analysis/spikes/spike-NNN-<topic>.md`

---

### Mode 5: Estimation

Produce detailed effort breakdowns per initiative: hours by activity, team composition required, risk buffer, and confidence level. Rolls up to a total roadmap estimate.

**Invocation:** `"Estimate effort"` / `"Break down effort for initiative [N]"`

**Output:** `roadmaps/analysis/metrics/effort-estimation.md`

---

### Mode 6: PDF Export

Package roadmap documents into professional PDF reports for stakeholder communication. Three packages: Executive (10–15 pages), Technical (50–100 pages), Initiative-specific.

**Invocation:** `"Export roadmap to PDF"` / `"Create executive roadmap package"`

**Output:** `roadmaps/pdf/<package>-YYYY-MM-DD.pdf`

---

### Mode 7: Team & Financial Planning

Analyse roadmap complexity to determine required team composition, roles, seniority levels, and allocation per phase. Optionally produce cost estimates and AI augmentation recommendations.

**Invocation:** `"Plan team for roadmap"` / `"What team do we need?"` / `"Estimate roadmap cost"`

**Output:** `roadmaps/team-planning/team-composition.md`, `roadmaps/team-planning/hiring-plan.md`

---

## Principles

- **One mode at a time** — offer the mode menu, run what's requested, offer next step
- **Ask before expanding all** — expanding every initiative is expensive; confirm which ones
- **Spikes before estimates** — high-uncertainty items need a spike before credible estimation
- **Separate executive from technical** — PDF exports and team plans exist in two audiences; don't conflate them
- **Estimates include confidence** — never present a number without its confidence level and assumptions
